# قاعدة البيانات التجريبية الاحترافية

هذه المرحلة مخصصة لبيئة التطوير المحلية فقط. تضيف أمرًا محميًا يعيد بناء قاعدة البيانات بالكامل ويحمّل بيانات مترابطة تغطي لوحة الإدارة، التقارير، الصلاحيات، نطاقات الوصول، وواجهات تطبيق Flutter.

## تحذير مهم

الأمر `demo:reset --apply` يحذف جميع الجداول والبيانات الحالية باستخدام `migrate:fresh` ثم يعيد إنشاءها. يمنع الأمر التنفيذ عندما تكون `APP_ENV=production`، ويعمل فقط في بيئة `local` أو `testing`.

## الفحص دون تعديل

```powershell
php artisan demo:reset --check
```

يعرض الأمر:

- بيئة التطبيق والاتصال وقاعدة البيانات المستهدفة.
- أعداد البيانات الحالية التي ستُحذف.
- ملخص البيانات التجريبية التي ستُنشأ.
- لا ينفذ أي حذف أو كتابة.

## التطبيق

```powershell
php artisan demo:reset --apply
```

سيطلب الأمر تأكيدًا صريحًا قبل الحذف. بعد المراجعة يمكن تجاوز السؤال في البيئة المحلية فقط:

```powershell
php artisan demo:reset --apply --force
```

## محتوى البيانات

- 4 مناطق و5 خطوط توزيع، بينها خط نشط بلا حركة لاختبار تقرير الأداء.
- 4 سيارات و7 مستودعات، تشمل المستودع الرئيسي والمبرد والاحتياط ومستودعات السيارات.
- 20 عميلًا بأنماط نقدي وجزئي وآجل، مع حالات متأخرة وحالات مسددة وجديدة.
- 15 منتجًا و6 تصنيفات و5 وحدات.
- أرصدة افتتاحية وتكاليف متوسط مرجح وحركات تحويل وبيع ومرتجعات.
- دفعة منتهية، دفعة دون صلاحية، ودفعات قريبة وآمنة لتغطية تقرير مخاطر الصلاحية.
- تحميلات معتمدة ومسودة.
- فواتير معتمدة ومسودة وملغاة.
- تحصيلات نقدية وتحويلات وشيكات ودفعات أخرى، بينها تحصيل على الحساب.
- مرتجعات معتمدة ومسودة.
- مصاريف معتمدة ومعلقة ومرفوضة.
- إغلاقات يومية معتمدة ومسودة وفروق صندوق وجرد متنوعة.
- نطاقات مباشرة للمشرف وأمين المستودع، ونطاقات مشتقة لحسابات Flutter.
