# نطاقات الوصول إلى البيانات — Role Data Scopes

تكمّل هذه المرحلة نظام RBAC الموجود في commit `e24247a`.

نظام RBAC يحدد **ماذا يستطيع المستخدم أن يفعل**، بينما هذه المرحلة تحدد **أي سجلات يستطيع رؤيتها أو تعديلها**.

## الأدوار ونطاقاتها

### Super Admin وManager وAccountant

هذه الأدوار تحتفظ بنطاق بيانات عام، مع بقاء كل عملية محكومة بصلاحيات RBAC الخاصة بها.

- `super_admin`: وصول كامل.
- `manager`: وصول تشغيلي عام، مع بقاء حماية حسابات Super Admin.
- `accountant`: وصول مالي عام وفق الصلاحيات المالية الممنوحة له.

### Supervisor

يُدار نطاق المشرف من شاشة المستخدمين عبر:

- المناطق.
- خطوط التوزيع الإضافية.
- السيارات الإضافية.
- المستودعات الإضافية.

النطاقات **تراكمية**:

- المنطقة تمنح خطوطها وعملاءها وسيارات الخطوط.
- الخط يمنح منطقته وسيارته وفريقه.
- السيارة تمنح الخطوط المرتبطة بها ومستودع السيارة.
- المستودع المرتبط بسيارة يمنح السيارة وخطوطها.
- المستودعات المستخدمة فعليًا في عمليات الخط تصبح جزءًا من النطاق الفعلي لضمان اكتمال العرض والطباعة.
- إسناد المستودع يمنح صلاحية العمل المخزني والتحقق من مستودعات العملية، لكنه لا يوسّع رؤية المبيعات إلى خطوط أخرى تستخدم المستودع نفسه.

### Warehouse Keeper

يُسند إلى أمين المستودع مستودع واحد أو أكثر.

يستطيع رؤية:

- المستودعات المسندة.
- أرصدتها وحركاتها.
- أوامر التحميل التي يكون أحد طرفيها مستودعًا مسندًا.
- السيارات والخطوط المرتبطة بمستودعات السيارات المسندة.
- تقارير المخزون والتحميل والصلاحية ضمن النطاق.

عرض أمر تحميل يكفيه ارتباط أحد المستودعين، لكن الاعتماد أو التعديل يحتاج أن يكون **مستودعا المصدر والوجهة** ضمن نطاقه.

### Sales Representative وDriver

لا تُدار نطاقاتهما يدويًا من شاشة المستخدمين.

يُشتق النطاق من:

1. حساب المستخدم المرتبط بسجل الموظف.
2. نوع الموظف المطابق للدور.
3. خطوط التوزيع المسندة للموظف.
4. السيارة والمستودعات وفريق الخط والعملاء والعمليات المرتبطة.

إذا لم يرتبط الحساب بموظف مطابق أو لم يُسند الموظف إلى خط، يكون النطاق فارغًا افتراضيًا.

## قاعدة الأمان

الأدوار المقيدة تستخدم قاعدة:

> لا تعيين = لا بيانات

لا تعتمد الحماية على إخفاء عناصر القائمة فقط. يتم تطبيق النطاق على:

- استعلامات Eloquent عبر Global Scope.
- Laravel Policies على مستوى السجل.
- إجراءات التأكيد والاعتماد والإلغاء والتعديل.
- الحفظ عبر Observer يمنع تمرير معرفات خارج النطاق.
- خيارات النماذج والعلاقات في Filament.
- Route Model Binding والطباعة.
- Dashboard.
- التقارير وخدمات الكاش.
- أي API يستخدم النماذج والخدمات نفسها مستقبلًا.

## جداول التعيين

```text
user_area_scopes
user_route_scopes
user_vehicle_scopes
user_warehouse_scopes
```

كل جدول يستخدم مفاتيح أجنبية حقيقية، وقيد uniqueness، وحذفًا متسلسلًا عند حذف المستخدم أو العنصر المرتبط.

## الحفاظ على السلوك الحالي

أثناء Migration:

- يحصل كل مشرف موجود حاليًا على جميع المناطق وجميع المستودعات.
- يحصل كل أمين مستودع موجود حاليًا على جميع المستودعات.

الهدف هو منع انقطاع وصول الحسابات التجريبية الحالية فور الترحيل. بعد التحقق يمكن تضييق التعيينات من شاشة المستخدمين.

## ملاحظات تشغيلية

- المنتجات والوحدات والتصنيفات تبقى بيانات مرجعية عامة لمن يملك صلاحية عرضها.
- العمليات الداخلية غير المرتبطة بطلب مستخدم، مثل أوامر الصيانة أو أوامر Console، تعمل دون نطاق مستخدم.
- الكاش الثابت في Dashboard وتقارير العملاء والخطوط يتضمن بصمة النطاق، لمنع مشاركة نتائج مستخدم مع مستخدم آخر.
- النطاق الفعلي يُحسب مرة واحدة في دورة الطلب ويُخزن مؤقتًا داخل الخدمة.
