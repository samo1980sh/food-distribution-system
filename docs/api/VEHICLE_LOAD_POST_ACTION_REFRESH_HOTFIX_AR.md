# إصلاح تحديث أمر تحميل المركبة بعد الإجراءات

يعالج هذا الإصلاح بقاء بيانات المسودة القديمة ظاهرة داخل صفحة تفاصيل أمر التحميل بعد تنفيذ الاعتماد أو الإلغاء.

## السلوك بعد الإصلاح

- يتم تحديث سجل أمر التحميل نفسه مباشرة بعد نجاح الاعتماد أو الإلغاء.
- تظهر الحالة الجديدة وإجمالي التكلفة والتشغيلات وتواريخ الصلاحية وبيانات المعتمد دون الحاجة إلى `Ctrl + F5`.
- لا يضيف الإصلاح أي حركة مخزون جديدة ولا يعيد تنفيذ الاعتماد.
- يبقى احتساب التكلفة داخل `VehicleLoadService`، بينما يقتصر هذا الإصلاح على مزامنة نسخة السجل المعروضة في Filament.
- تم تعزيز اختبار FEFO للتحقق من تكلفة كل سطر ومن إجمالي تكلفة أمر التحميل.
