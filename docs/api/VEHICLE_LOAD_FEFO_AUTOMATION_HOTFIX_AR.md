# إصلاح التحميل التلقائي حسب الدفعات — Phase 10

يُبقي نموذج أمر تحميل المركبة بسيطًا: يختار المستخدم المنتج والكمية فقط.

عند الاعتماد يقوم النظام تلقائيًا بما يلي:

- يجمع الكمية المطلوبة لكل منتج.
- يقرأ الأرصدة الموجبة من المستودع المصدر.
- يستبعد الدفعات المنتهية قبل تاريخ التحميل.
- يطبق FEFO باختيار أقرب تاريخ صلاحية أولًا.
- يقسم الكمية على أكثر من دفعة عندما لا تكفي دفعة واحدة.
- يحفظ رقم التشغيلة وتاريخ الصلاحية على بنود أمر التحميل الناتجة.
- ينقل المخزون مرة واحدة لكل تخصيص ويحافظ على تكلفة كل دفعة.
- يعكس الإلغاء نفس التخصيصات إلى المستودع المصدر.

لا يحتاج المستخدم إلى إدخال رقم التشغيلة أو تاريخ الصلاحية يدويًا.
