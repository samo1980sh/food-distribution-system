# Phase 10 — استلام أوامر تحميل المركبات من التطبيق

## الهدف

إتاحة أوامر التحميل المعتمدة ومخزون المركبة للمستخدم الميداني، مع تسجيل استلام كامل أو فروقات استلام عبر مسار REST أو Offline Push آمن.

## المسؤوليات

- المستودع أو الإدارة ينشئان أمر التحميل ويعتمدانه.
- اعتماد أمر التحميل هو المسؤول عن حركة المخزون الحالية من المستودع إلى مستودع المركبة.
- التطبيق الميداني لا ينشئ حركة مخزون جديدة عند تأكيد الاستلام.
- تأكيد الاستلام يوثق الواقع الميداني فقط، ولا يغير الكميات المعتمدة أو تكاليفها.
- أي فرق مسجل يحتاج معالجة إدارية مستقلة عبر حركة أو تسوية مخزون مناسبة.

## العقد

`POST /api/v1/operational/vehicle-loads/{vehicleLoad}/acknowledge`

```json
{
  "handover_status": "received",
  "notes": "",
  "items": [
    {
      "id": 10,
      "received_quantity": "12.000",
      "note": ""
    }
  ]
}
```

القيم المدعومة لـ`handover_status`:

- `received`: جميع الكميات المستلمة مطابقة للمحمّل.
- `discrepancy`: توجد فروقات فعلية أو ملاحظة موثقة.

## Offline Push

- entity: `vehicle_loads`
- action: `acknowledge`
- record_id: رقم أمر التحميل على الخادم
- base_version: نسخة السجل المسحوبة آخر مرة
- payload: نفس حمولة مسار REST

تخضع العملية لنفس قواعد Idempotency وتعارض النسخة المستخدمة في بقية العمليات المالية والمخزنية.
